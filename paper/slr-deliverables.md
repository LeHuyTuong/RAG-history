# Systematic Literature Review Deliverables

## 1. Executive Summary

This review found strong direct evidence for Vietnamese general and multi-domain retrieval, strong direct evidence for Vietnamese legal retrieval, moderate indirect evidence from multilingual retrieval benchmarks and model papers, and insufficient direct evidence for Vietnamese historical document retrieval. No direct benchmark evidence was found for Vietnamese historical document retrieval. The candidate recommendation is therefore based on indirect Vietnamese/general/multilingual retrieval evidence and must be empirically validated on historical PDFs or page-level passages before any "best model" claim is made.

Best practical first candidates:

1. Hybrid BM25 + BGE-M3: first candidate because it combines lexical matching, dense retrieval, sparse retrieval, and optional multi-vector scoring.
2. multilingual E5 large/instruct: strong multilingual baseline.
3. gte-multilingual-base: efficient candidate baseline with long context and dense/sparse support, supported mainly by model-card evidence plus VN-MTEB inclusion.
4. Vietnamese-tuned BGE/E5/SBERT/PhoBERT bi-encoder: promising if fine-tuned on historical or legal-style relevance pairs.
5. LaBSE: useful cross-lingual baseline, but weaker evidence for modern Vietnamese passage retrieval.

## 7. OCR and PDF Processing Guidance

Do Vietnamese history PDFs need OCR? Not always. If the PDF has selectable text and Vietnamese diacritics are preserved, OCR is unnecessary. OCR is needed for scanned image-only pages, mixed scanned pages, old font encodings that extract incorrectly, and degraded images.

How to detect OCR need:

1. Attempt text selection or extraction on several pages.
2. Check Vietnamese diacritics, names, dates, page headers, and footnotes.
3. Compare extracted text with page images for random pages.
4. Mark each page as text, scan, mixed, or extraction-failed.

How OCR errors affect embedding retrieval: semantic noise changes words and names; formatting noise breaks tables, footnotes, reading order, and page boundaries. Both can reduce dense similarity and BM25 matching.

How to preserve page numbers: store document ID, page number, original PDF page index, section heading, extraction method, OCR confidence, and chunk offsets with every chunk. Never merge pages without retaining source-page metadata.

Recommended pipeline:

1. PDF inspection.
2. Text extraction.
3. OCR only if needed.
4. Vietnamese Unicode and whitespace normalization.
5. Page-aware chunking.
6. Metadata preservation.
7. Embedding with multiple candidates.
8. Hybrid retrieval.
9. Reranking if needed.
10. Answer generation with page citations.

OCR decision table:

| PDF condition | OCR needed? | Recommended action | Risk |
| --- | --- | --- | --- |
| Selectable text PDF | No | Extract text with PyMuPDF/pdfplumber and verify accents | Low |
| Scanned image PDF | Yes | Use OCR and inspect samples | High |
| Mixed text and scanned pages | Partial | OCR only scanned pages | Medium |
| PDF with tables/footnotes | Maybe | Extract text and manually inspect layout | Medium |
| Old Vietnamese fonts | Maybe | Test extraction and OCR quality | High |
| Poor scan quality | Yes but unreliable | OCR plus manual sample correction | High |
| Historical documents with names/dates | Maybe | Validate entities, dates, and page metadata | High |
